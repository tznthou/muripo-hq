挑戰結束了，但我的腦袋還沒停下來。

## 趕工的代價

這三十多天，每天都在趕、趕、趕。

原本給自己設定的是：每個專案在特定時數內完成。結果呢？**幾乎每天都超時**。

計畫永遠趕不上變化，尤其是當你一邊學習、一邊創作的時候。

## 累積的債務

現在回頭看這 32 個 repo，有些東西讓我有點⋯⋯尷尬：

- **README 格式不統一**：每天趕工的結果就是，每個專案的 README 長得都不太一樣。有些寫得很詳細，有些就⋯⋯勉強能看。
- **前半期沒做 Code Review**：後半期的 Web 專案我都有讓 Claude 做 Code Review 和資安檢查，但前半期的專案？完全沒有。
- **資安檢查不完整**：OWASP Top 10 的那些東西，前半期的專案應該也要補一下。

## 還有什麼可以做？

仔細想想，收尾工作還真不少：

1. **統一 README 格式**：建立一個模板，把所有專案的 README 都整理成一致的格式
2. **補做 Code Review**：前半期的專案也該讓 AI 審一審，看看有沒有明顯的問題
3. **資安健檢**：Web 類型的專案，特別是有用戶輸入的部分，都應該做個基本的安全檢查
4. **效能優化**：有些專案跑起來其實有點慢，可以花時間優化一下
5. **文件補充**：有些專案的使用說明寫得太簡略，可以補充更多範例

## 也許該寫個 README 檢查器？

等等，我突然有個想法。

既然每個專案的 README 格式都不統一，而且我有 32 個 repo 要檢查⋯⋯

**也許我應該寫一個程式來檢查 README.md？**

可以設定一些規則：
- 是否有專案標題
- 是否有安裝說明
- 是否有使用範例
- 是否有 License 說明
- 格式是否符合特定模板

然後批次掃描所有 repo，產出一份報告告訴我哪些 README 需要補強。

嗯，這聽起來可以是下一個專案。

## 學無止境

這三十多天，我學到的東西比我預期的還要多。

但每次學到新東西，就會發現還有更多不懂的。

就像開了一扇門，門後面還有無數扇門。

## AI Coding 太好玩了

說真的，如果沒有 AI 幫忙，這 32 個專案我大概連一個都完不成。

不是謙虛，是實話。

這些技術棧、這些框架、這些工具——光是自己摸索就不知道要花多少時間。

有了 AI，遇到不懂的概念，問一下就有答案。卡住了不知道怎麼繼續，討論一下就有方向。想嘗試新技術但怕踩坑，先問問有什麼注意事項。

**這種學習加速的感覺，真的太棒了。**

## 2025 年結束了

今天是 12 月 31 日，2025 年的最後一天。

這個挑戰結束了，但我的學習之旅才剛開始。

那些收尾工作，就當作是 2026 年的待辦事項吧。
