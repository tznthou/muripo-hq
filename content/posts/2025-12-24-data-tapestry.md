這個專案是從 Day 19 的 Stargazer Galaxy 延伸出來的。

當時我們用 GitHub Action 把 Stargazer 變成銀河中的星星，那是「空間」上的堆疊。做完之後我就在想：GitHub Action 還能有什麼樣的可能性？

然後我看到了 Flat Data。

## Flat Data：讓數據自己說話

Flat Data 是 GitHub Next 團隊做的一個工具。它的概念很簡單：每天自動抓取一個 API 的數據，然後 commit 到你的 repo 裡。

聽起來沒什麼特別的，但仔細想想，這代表你的 Git History 本身就變成了一個時間序列資料庫。每一次 commit 就是一個時間切片。

這讓我想到：如果每天抓取 GitHub 的趨勢數據，然後把它變成一條線，那經過一個月、一年，會變成什麼樣子？

## 為什麼是 GitHub 趨勢？

這 30 天的專案挑戰，我們幾乎都圍繞著 GitHub 打轉。

程式碼放在 GitHub、文件放在 GitHub、部署也靠 GitHub Actions。既然我們如此依賴這個平台，那觀察它的脈搏似乎是一件很自然的事。

每天有哪些專案在燃燒？哪些語言正在崛起？開源社群的注意力流向哪裡？

這些問題的答案，每天都在變化。

## 我不知道最後會變成什麼樣子

說實話，這是我覺得最有趣的部分。

我們把每一天的趨勢數據變成一條「經緯線」，織進一張 SVG 織錦裡。Python 是藍色、TypeScript 是深藍、Rust 是橘色。線的粗細代表熱度，波動代表變化。

但最後這張圖會長成什麼樣子？我不知道。

沒有人能百分之百預測未來的趨勢。也許可以猜，也許可以用模型去推測，但完全命中？絕對有困難。

所以我選擇不預測，而是記錄。

讓時間自己編織答案。

## 把數據變成黃金

既然每天都在抓取數據，不如讓它發揮更多價值。

所以我把當天的熱門專案直接更新到 README 上。這樣每次有人進來看織錦，順便也能看到昨天 GitHub 上最火紅的 repo 是什麼。

不用特地去找 GitHub Trending 頁面，這裡就有每日精選。

數據不該只是被收集然後放在那裡。既然抓了，就讓它活起來。

## 一起保持好奇心

這個專案會持續運作下去。每天台灣時間 00:00，它會自動抓取數據、織入新的一條線。

一個月後回來看，會看到 30 條線。一年後回來看，會看到 365 條線交織成的圖案。

那會是什麼樣子？我也很好奇。

如果你也想知道答案，歡迎跟我一起觀察下去。
